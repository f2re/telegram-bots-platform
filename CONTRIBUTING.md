## How Can I Contribute?

### Reporting Bugs

- Check the existing issues to avoid duplicates
- Use a clear and descriptive title
- Include as many details as possible:
  - Steps to reproduce the issue
  - Expected behavior
  - Actual behavior
  - Environment details (OS, platform version, etc.)

### Suggesting Enhancements

- Check if the enhancement has already been suggested
- Provide a clear and detailed explanation of the proposed enhancement
- Include specific use cases

### Pull Requests

- Fork the repository
- Create a new branch for your feature or bug fix
- Follow the style guidelines
- Add or update documentation as needed
- Submit your pull request

## Development Setup

1. Fork and clone the repository:

```bash
git clone https://github.com/your-username/telegram-bots-platform.git
cd telegram-bots-platform
```

2. Make sure you have the required dependencies installed on a Debian 12 system

3. Run the setup script for testing:

```bash
./setup-server.sh
```

## Pull Request Process

1. Ensure your code follows the style guidelines
2. Update documentation as needed
3. Test your changes thoroughly
4. Squash your commits if there are many small ones
5. Submit your pull request with a clear description of the changes

## Style Guidelines

### Shell Scripts

- Use consistent indentation (4 spaces)
- Include comments for complex logic
- Follow POSIX compliance where possible
- Use descriptive variable names
- Include error handling where appropriate

### Configuration Files

- Maintain consistent formatting
- Include comments explaining important settings
- Follow established patterns in the project

## Questions

If you have any questions about contributing, please open an issue in the repository.
